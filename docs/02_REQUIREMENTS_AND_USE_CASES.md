# Requirements and Use Cases

## 1. 범위

이 문서는 PRISM Club MVP의 기능 요구사항, 비기능 요구사항, 유스케이스, 수용 기준을 정의한다.

이번 버전의 Club은 참여 기록/랭킹 앱이 아니라, 예능 콘텐츠와 오프라인 이벤트 경험을 주제별로 정리하고 이야기하는 지식형 커뮤니티 서비스다. 핵심 단위는 Topic Hub, 카테고리, 방, 지식 블록, 타임라인, 게시글, 댓글, 대댓글, 이벤트 카드, 레퍼런스 카드다.

## 2. 액터

| 액터 | 설명 |
| --- | --- |
| Guest | 가입 전 사용자. 공개 카테고리와 일부 글을 볼 수 있다. |
| Member | 일반 사용자. 글, 댓글, 대댓글, 좋아요, 저장, 방 생성, 신고를 할 수 있다. |
| Verified Participant | PRISM Events/Contenido 참가 기록이 확인된 사용자. 참가 인증 후기를 작성할 수 있다. |
| Event Host | 이벤트 기획자. 자기 이벤트 관련 질문에 답하거나 공지성 댓글을 남길 수 있다. |
| Verified Planner | 인증된 기획자/스튜디오 멤버. 기획자 전용 카테고리에 입장하고 모집/운영 글을 작성할 수 있다. |
| Room Owner | 방을 만든 사용자. 방 설명, 태그, 대표 레퍼런스, 대표 이벤트 카드를 관리한다. |
| Knowledge Contributor | Topic Hub의 정보 개선, FAQ 추가, 레퍼런스 정리를 제안하는 사용자 |
| Curator | Topic Hub의 지식 블록과 데이터 신호를 검수하고 정리하는 운영자 또는 신뢰 사용자 |
| Moderator | 커뮤니티 운영자. 방 규칙, 신고, 숨김, 제재를 관리한다. |
| Admin | 전체 운영자. 카테고리, 방, 이벤트 연동, 정책을 관리한다. |
| System | Events/Contenido, Notification, Search, Moderation 등 내부 시스템 |

## 5. 핵심 유스케이스

### UC-01 Topic Hub 탐색

| 항목 | 내용 |
| --- | --- |
| 액터 | Member |
| 선행 조건 | 카테고리와 Topic Hub가 존재 |
| 기본 흐름 | Club 홈 진입 → 연애 콘텐츠 선택 → 개요/핵심 정보/관련 이벤트/인기 레퍼런스/FAQ/관련 방 확인 |
| 예외 | 접근 권한이 필요한 Topic Hub면 인증 안내 표시 |
| 수용 기준 | Topic Hub에서 지식 블록과 관련 방이 함께 표시된다. |

### UC-02 Topic Hub 정보 기여

| 항목 | 내용 |
| --- | --- |
| 액터 | Knowledge Contributor, Curator |
| 선행 조건 | 사용자가 가입되어 있고 Topic Hub가 존재 |
| 기본 흐름 | Topic Hub → 정보 개선 제안 → 내용/근거 레퍼런스 입력 → Curator 검토 → 승인 시 지식 블록 반영 |
| 예외 | 근거가 부족하면 보완 요청 또는 거절 |
| 수용 기준 | 승인된 기여는 지식 블록에 반영되고 변경 이력이 남는다. |

| 선행 조건 | 사용자가 PRISM Events/Contenido 이벤트 참가 기록을 가지고 있음 |
| 기본 흐름 | 글쓰기 → 이벤트 검색 → 이벤트 카드 첨부 → 후기 작성 → 인증 후기 태그 표시 |
| 예외 | 참가 기록이 없으면 일반 토론 글로 작성되며 인증 태그는 표시되지 않음 |
| 수용 기준 | 게시글에 이벤트 카드와 인증 후기 표시가 함께 노출된다. |

## 6. MVP 수용 기준

1. 사용자는 카테고리 진입 시 Topic Hub에서 정리된 정보와 관련 방을 함께 볼 수 있다.
2. 사용자는 Topic Hub의 정보 개선을 제안할 수 있다.
3. Curator/Admin은 지식 기여를 검수하고 반영할 수 있다.
4. 사용자는 카테고리 안에 새 방을 만들 수 있다.
5. 사용자는 방 타임라인에서 글을 읽고 작성할 수 있다.
6. 사용자는 댓글과 대댓글을 작성할 수 있다.
7. 사용자는 Events/Contenido 이벤트 카드를 게시글이나 방 대표 자료에 첨부할 수 있다.
8. 사용자는 예능 프로그램/영상/룰 링크를 레퍼런스 카드로 공유할 수 있다.
9. 사용자는 참가자 커뮤니티와 기획자 커뮤니티를 구분해 탐색할 수 있다.
10. 인증된 기획자는 기획자 전용 카테고리에서 모집/아이디어/운영 글을 작성할 수 있다.
11. 사용자는 Topic Hub, 카테고리, 방, 게시글, 이벤트, 레퍼런스, 모집 글을 검색할 수 있다.
12. 운영자는 카테고리/방을 관리하고 신고된 글을 처리할 수 있다.
13. 신고/숨김/삭제/제재 조치는 audit log에 남는다.
