## 1. 제품 정의

PRISM Club은 예능 콘텐츠, 오프라인 이벤트, 놀이 경험, 프로그램 레퍼런스를 주제별로 모아 이야기하는 지식형 커뮤니티 서비스다.

운영자가 만든 큰 카테고리는 기본 골격으로 존재한다. 사용자는 “연애 콘텐츠”, “추리 게임”, “MT/워크숍”, “런닝맨식 미션”, “오프라인 어몽어스” 같은 카테고리에 들어가고, 그 안에서 더 디테일하거나 신박한 주제의 방을 직접 만들 수 있다. 각 방은 트위터/스레드처럼 타임라인 방식으로 글이 흐르고, 게시글에는 PRISM Events/Contenido에서 진행 예정이거나 진행했던 이벤트 정보, 참가 후기, 게임 기록, 사진, 영상 링크, 예능 프로그램 레퍼런스, 아이디어를 연결할 수 있다.

단, 카테고리는 단순히 주제명 하나만 있는 게시판이 아니다. 사용자가 주제를 클릭하면 그 주제의 개요, 핵심 개념, 관련 이벤트, 인기 레퍼런스, 자주 묻는 질문, 최근 반응, 관련 방, 기획 팁 같은 디테일한 정보가 모인 Topic Hub로 들어간다. 커뮤니티 대화는 이 정보 허브 위에 쌓이는 레이어이며, 장기적으로는 사용자들이 함께 만드는 거대한 오프라인 엔터테인먼트 지식 데이터베이스가 된다.

제품 감각은 게시판보다 트위터/스레드에 가깝다. 빠르게 쓰고, 이어서 답글을 달고, 대댓글로 맥락을 깊게 만들며, 특정 주제 안에서 경험과 정보를 축적한다.

또한 Club은 참가자 커뮤니티와 기획자 커뮤니티를 분리한다. 참가자 커뮤니티는 이벤트 반응, 후기, 정보공유, 소셜링에 집중하고, 기획자 커뮤니티는 아이디어 소통, 스태프 모집, 특정 스튜디오 운영 정보, 레퍼런스 공유, 이벤트 포맷 검토를 다룬다. 일부 카테고리와 방은 인증된 기획자나 스튜디오 멤버만 입장할 수 있다.

## 2. PRISM 전체 맥락

기획안은 PRISM을 7개 색깔 서비스가 연결된 오프라인 엔터테인먼트 생태계로 정의한다.

| 색 | 서비스 | 역할 |
| --- | --- | --- |
| 빨강 | PRISM Events / Contenido | 이벤트 탐색, 신청, 결제, 참여 |
| 주황 | PRISM Studio | 기획자용 이벤트 생성과 운영 |
| 노랑 | PRISM MC | AI 사회자, 진행 멘트, 타이머, 투표 |
| 초록 | PRISM Play / MOYA | 실시간 위치 기반 오프라인 게임 |
| 파랑 | PRISM Partner | 장소, 소품, 촬영, 청소, 인력 연결 |
| 남색 | PRISM Maker | AI 오프라인 게임 제작 도구 |
| 보라 | PRISM Club | 주제 지식 허브, 커뮤니티, 이벤트 경험 공유, 레퍼런스 아카이브 |

Club은 PRISM 생태계에서 “경험이 이야기와 지식으로 바뀌는 곳”이다. Events는 참여 입구이고, Play는 현장 놀이이고, MC는 진행 도우미라면, Club은 사람들이 그 경험을 다시 해석하고 공유하며 다음 콘텐츠를 찾고, 함께 데이터를 정리하는 장소다.

## 3. 핵심 가치

1. 주제별 지식 허브를 만든다: 카테고리마다 개요, 핵심 정보, 레퍼런스, 이벤트, FAQ, 관련 방을 모은 Topic Hub를 제공한다.
2. 주제별 커뮤니티를 만든다: 예능 콘텐츠와 오프라인 놀이 카테고리마다 독립적인 대화 공간을 제공한다.
3. 유저가 방을 만든다: 큰 카테고리 안에서 사용자가 직접 세부 방을 만들고, 그 방의 주제와 참고 자료를 쌓는다.
4. 집단 지식을 축적한다: 사용자의 글, 댓글, 후기, 이벤트 반응, 레퍼런스가 주제별 데이터로 정리된다.
5. 참가자와 기획자를 분리한다: 참가자 커뮤니티와 기획자 전용 커뮤니티를 나누어 정보의 성격과 접근 권한을 다르게 둔다.
6. 이벤트 경험을 축적한다: PRISM Events/Contenido의 진행 예정/진행 완료 이벤트 정보, 후기, 기록을 글과 댓글 맥락으로 불러온다.
7. 레퍼런스를 공유한다: 예능 프로그램, 게임 룰, 미션 아이디어, 진행 방식, 촬영 구도 등을 링크/글/카드로 모은다.
8. 빠른 소통을 가능하게 한다: 타임라인, 짧은 게시글, 답글, 대댓글, 인용, 좋아요, 저장을 제공한다.
9. 정보 탐색을 돕는다: 사용자가 “어떤 콘텐츠가 재밌었는지”, “어떤 이벤트가 믿을 만한지”, “어떻게 기획하면 좋은지”, “어디서 스태프를 구할지” 찾을 수 있게 한다.
10. PRISM의 재방문 이유를 만든다: 이벤트가 끝난 뒤에도 커뮤니티 대화가 이어지고, 그 대화가 다음 이벤트 참여나 기획으로 연결된다.

## 5. MVP 범위

MVP 시나리오:

사용자가 “연애 콘텐츠”를 클릭한다. 사용자는 먼저 Topic Hub에서 연애 콘텐츠의 개요, 인기 포맷, 진행 예정 이벤트, 진행 완료 이벤트 반응, 많이 저장된 예능 레퍼런스, 자주 묻는 질문, 관련 방을 확인한다. 이후 기본 방에서 소개팅형 이벤트 후기를 읽고, “환승연애식 오프라인 토크 게임”이라는 새 방을 만든다. 이 방에는 예능 프로그램 레퍼런스와 진행 예정 이벤트 카드를 대표 자료로 붙인다. 다른 사용자들은 그 방의 타임라인에 글을 쓰고 대댓글로 “이 포맷을 오프라인 이벤트로 만들면 어떻게 바꿀지” 토론한다.

기획자는 기획자 커뮤니티의 “스태프/스튜디오” 카테고리에 들어간다. 인증된 기획자만 접근 가능한 방에서 특정 스튜디오의 주말 이벤트 스태프 모집 글을 올리고, 지원자는 댓글과 대댓글로 역할, 시간, 페이, 준비물을 확인한다.

| 현장 스태프 | 참여 가능한 현장 운영 일을 찾고 정보를 확인한다 | 모집 필터, 역할/시간/페이 정보, 댓글 문의 |
| 참가 예정자 | 이벤트가 믿을 만한지, 어떤 분위기인지 확인한다 | 실제 참가 후기, FAQ, 운영자 답변 |
| 운영자 | 커뮤니티 품질과 안전을 유지한다 | 신고 처리, 방 규칙, 숨김/제재 |

## 7. 성공 지표

| 지표 | 의미 | MVP 목표 예시 |
| --- | --- | --- |
| 글 작성률 | 사용자가 구경을 넘어 참여하는지 | 가입자 중 주 1회 글 작성 15% |
| 댓글 전환율 | 대화가 이어지는지 | 게시글당 평균 댓글 2개 이상 |
| 이벤트 카드 첨부율 | Events/Contenido와 연결되는지 | 이벤트 후기 글의 60% 이상 |
| 레퍼런스 저장 수 | 콘텐츠 아이디어 아카이브가 쌓이는지 | 주당 50개 이상 |
| Topic Hub 체류율 | 주제 정보 허브가 정보 탐색에 쓰이는지 | 카테고리 진입 세션 평균 2분 이상 |
| 지식 기여 수 | 사용자가 지식 데이터 축적에 참여하는지 | 주당 20건 이상 |
| 유저 생성 방 활성률 | 사용자가 직접 만든 방이 살아나는지 | 생성 방의 40%가 7일 내 글 5개 이상 |
| 기획자 커뮤니티 참여율 | 기획자 전용 공간의 실사용 여부 | 인증 기획자 주간 활동률 30% |
| 모집 글 전환율 | 스태프 모집이 실제 문의로 이어지는지 | 모집 글당 댓글/문의 3개 이상 |
| 검색 후 체류 | 정보 탐색 가치가 있는지 | 검색 세션 평균 3분 이상 |
| 신고 처리 SLA | 커뮤니티 신뢰 | 24시간 내 90% 처리 |
| 이벤트 재유입 | Club에서 Events로 넘어가는지 | 이벤트 카드 클릭률 10% 이상 |

## 8. 아직 결정해야 할 질문

1. 유저 생성 방을 즉시 공개할지, 신뢰도/스팸 체크 후 공개할지 정해야 한다.
2. 기획자 전용 카테고리 입장 조건을 어떻게 인증할지 정해야 한다. 예: Studio 계정, 이벤트 호스트 이력, 운영자 승인.
3. Topic Hub의 정보 블록을 누가 편집할 수 있을지 정해야 한다. 예: 운영자만, 신뢰 사용자 제안, 기획자 승인.
4. 사용자 활동에서 어떤 데이터 신호를 Topic Hub에 자동 반영할지 정해야 한다.
5. 이벤트 후기는 참가 인증 사용자만 쓸 수 있게 할지, 일반 토론과 분리할지 결정해야 한다.
6. 대댓글 깊이는 제한할지, 스레드처럼 계속 이어지게 할지 정해야 한다.
7. 예능 프로그램 레퍼런스 공유 시 저작권/썸네일/인용 범위 정책이 필요하다.
8. 스태프 모집 글에 필요한 필수 정보와 안전 정책을 정해야 한다. 예: 역할, 시간, 장소, 보상, 연락 방식.
9. 닉네임 기반 익명성을 유지할지, 이벤트 참가 기록과 연결된 신뢰 표시를 둘지 결정해야 한다.
