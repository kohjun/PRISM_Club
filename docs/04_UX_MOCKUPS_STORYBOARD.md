# UX Mockups and Storyboard

## 1. 정보 구조

PRISM Club MVP는 하단 탭 4개로 시작한다.

| 탭 | 목적 |
| --- | --- |
| 홈 | 관심 카테고리의 새 글, 인기 토론, 이벤트 관련 글 |
| 카테고리 | 참가자/기획자 커뮤니티와 Topic Hub 탐색 |
| 검색 | Topic Hub, 게시글, 이벤트 카드, 레퍼런스, 방 검색 |
| 프로필 | 내가 쓴 글/댓글, 저장한 글, 관심 카테고리 |

관리 기능은 별도 Next.js Admin Web에서 제공한다.

## 4. 스토리보드: Topic Hub에서 지식과 대화가 쌓이는 흐름

| 장면 | 사용자 행동 | 시스템 반응 |
| --- | --- | --- |
| 1 | 사용자가 Club 홈에서 연애 콘텐츠를 선택한다. | Topic Hub가 열리고 개요, 핵심 정보, 이벤트, 레퍼런스, 관련 방을 보여준다. |
| 2 | 사용자가 핵심 정보에서 빠진 내용을 발견한다. | 정보 개선 제안 화면을 연다. |
| 3 | 사용자가 근거 레퍼런스와 함께 정보 개선을 제안한다. | 기여 내역이 검수 큐에 등록된다. |
| 4 | 사용자가 “환승연애식 오프라인 토크 게임” 방을 만든다. | 방 이름, 설명, 태그, 대표 자료를 저장한다. |
| 5 | 사용자가 방 대표 자료로 예능 레퍼런스와 예정 이벤트 카드를 붙인다. | 방 상단에 대표 자료가 고정된다. |
| 6 | 사용자가 글쓰기를 누르고 이벤트를 검색한다. | Events/Contenido에서 관련 이벤트 목록을 불러온다. |
| 7 | 사용자가 참가한 이벤트를 선택한다. | 이벤트 카드가 글에 첨부되고 참가 인증 여부를 확인한다. |
| 8 | 사용자가 후기를 작성해 게시한다. | 방 타임라인과 이벤트 관련 글 목록에 게시글이 표시된다. |
| 9 | 다른 사용자가 댓글과 대댓글을 단다. | 글 작성자에게 알림이 발송되고 쟁점 신호가 쌓인다. |
| 10 | Curator가 정보 개선 제안을 승인한다. | Topic Hub의 지식 블록이 업데이트된다. |
| 11 | 기획자가 Topic Hub와 토론을 읽고 다음 이벤트 아이디어를 얻는다. | 저장/인기 지표가 쌓이고, 운영자는 트렌드를 확인한다. |

| 화면 | Empty | Loading | Error | Success |

| 방 만들기 | 대표 자료 없음 | 생성 spinner | 중복/스팸 안내 | 방 생성 완료 |
| 방 타임라인 | 첫 글 작성 CTA | post skeleton | 필터 재시도 | 글 목록 |

| 게시글 상세 | 댓글 없음 | reply skeleton | 댓글 재조회 | 댓글/대댓글 |
| 검색 | 최근 검색/추천 키워드 | result skeleton | 검색 재시도 | 결과 목록 |
| 관리자 | 신고 없음 | table skeleton | 처리 실패 | 신고 큐 |

## 6. 디자인 메모

1. Club은 커뮤니티 앱이므로 홈부터 콘텐츠가 보여야 한다. 설명형 랜딩보다 바로 읽고 쓰는 화면이 좋다.
2. 기본 테마는 화이트모드다. 배경은 흰색/옅은 회색을 중심으로 두고, Club의 보라색은 탭 선택, 주요 버튼, 링크, 강조 배지에 제한적으로 쓴다.
3. 권장 컬러 방향은 `background #FFFFFF`, `surface #F7F7FB`, `text #17151F`, `muted #6F6A7A`, `club primary #7C3AED`, `club soft #F1EAFE`이다.
4. Topic Hub는 정보가 먼저 보이고, 관련 방과 타임라인은 그 아래로 이어져야 한다.
